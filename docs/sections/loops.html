Python Fundamentals #3

Loops & Control Flow

For, while, enumerate, comprehensions, try-except - Flow control mastery

Apa itu Loops & Control Flow?

üìö Definisi

Loops adalah struktur yang mengulang blok kode berkali-kali. Control Flow (Alur Kontrol) adalah mekanisme yang menentukan urutan kode - kode mana yang dijalankan, kapan dijalankan, dan berapa kali dijalankan.

Bayangkan sebagai "otak" pembuat keputusan program - loops untuk pengulangan, conditionals untuk pilihan, dan keduanya mengatur jalan cerita program.

‚úÖ Mengapa Loops & Control Flow Penting?

1. Prinsip DRY (Jangan Ulangi)

Daripada copy-paste kode 100x, loop sekali saja! Hemat baris dan kurangi error.

2. Dynamic Behavior

Bisa atur jumlah data yang tidak pasti. Proses 10 barang atau 10.000 barang kodenya sama.

3. Decision Making

Program harus memilih: jika tabrakan, balik arah. Control flow yang mengatur ini.

4. Real-time Systems

Aplikasi harus terus berjalan merespon user setiap saat, update layar terus menerus.

üîÑ Jenis-Jenis Loop di Python

For Loop

Iterasi urutan (jumlah diketahui)

for item in collection:
    process(item)

Gunakan saat: Kamu tahu apa yang mau diulang

While Loop

Ulangi sampai kondisi salah (jumlah tak tentu)

while condition:
    do_something()

Gunakan saat: Kamu tidak tahu berapa kali harus mengulang

üéØ Struktur Kontrol Alur

Pengandaian (if-elif-else)

Membuat keputusan berdasarkan kondisi

if ball_colliding:
    reverse_velocity()
elif out_of_bounds:
    reset_position()
else:
    continue_moving()
Exception Handling (try-except)

Menangani error tanpa bikin program crash

try:
    value = float(input)
except ValueError:
    show_error("Invalid input!")

üí° Tips Terbaik (Best Practices)

‚úÖ DO
  • ‚Ä¢ Use for for iterables
  • ‚Ä¢ Use enumerate() when need index
  • ‚Ä¢ Guard clauses (early return) to reduce nesting
  • ‚Ä¢ Pecah loop yang rumit jadi function kecil
‚ùå DON'T
  • ‚Ä¢ Don't modify list while iterating it
  • ‚Ä¢ Avoid infinite loops without break condition
  • ‚Ä¢ Don't nest loops >3 levels deep
  • ‚Ä¢ Don't use while True in GUI apps

üéì Analogi Alur Program

Bayangkan program sebagai resep: Loops = "aduk sampai rata", Conditionals = "jika terlalu kental, tambah air", Functions = "siapkan saus (lihat halaman 5)". Control flow memberitahu komputer BAGAIMANA cara menjalankan resepnya!

üéØ Guard Clauses (Early Return)

# app.py - Guard clause pattern
if not self.is_running:
    return  # Exit early if not running

# app.py - Another guard
if self.is_paused:
    self.animation_callback_id = self.root.after(50, self._run_loop)
    return  # Exit early

Pattern: Check fail conditions first, return early. Reduces nesting!

‚ö†Ô∏è Exception Handling Best Practices

  • ‚Ä¢ Catch SPECIFIC exceptions (ValueError) not generic Exception when possible
  • ‚Ä¢ Selalu beri tahu user saat error terjadi
  • ‚Ä¢ Use try-except for operations that can fail (file I/O, network, user input)

# app.py - Custom animation loop dengan after()
def _run_loop(self) -> None:
    """Loop utama simulasi (dipanggil berulang menggunakan after)."""
    if not self.is_running:
        return  # Exit guard

    if self.is_paused:
        # Tetap schedule check singkat saat pause
        self.animation_callback_id = self.root.after(50, self._run_loop)
        return  # Exit early

    # Move balls
    self.ball_1.move(TIME_STEP)
    self.ball_2.move(TIME_STEP)

    # Bounce off walls
    cw = self.canvas.winfo_width()
    ch = self.canvas.winfo_height()
    handle_wall_bounce(self.ball_1, cw, ch, PIXELS_TO_METERS)
    handle_wall_bounce(self.ball_2, cw, ch, PIXELS_TO_METERS)

    # Handle collision antara dua bola
    restitution = float(self.restitution_coefficient.get())
    force, finished_impulse = calculate_collision(
        self.ball_1, self.ball_2, restitution, self.contact_tracker
    )
    
    # Schedule next frame (recursive dengan after)
    self.animation_callback_id = self.root.after(
        int(TIME_STEP * 1000), self._run_loop
    )

üîÑ Event Loop Pattern

Instead of while True, GUI apps use root.after() to schedule function calls. This keeps UI responsive while running periodic updates.
